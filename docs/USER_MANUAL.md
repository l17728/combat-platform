# 作战平台用户手册

> 面向操作人员（攻关人员 / 责任田 / 运营）。线上：http://124.156.193.122:3001/

## 0. 通用表格操作

所有列表页面（攻关列表、全员名单、贡献录入、用户管理、配置中心、荣誉排行）支持以下表格交互：

- **列宽调整**：将鼠标移到任意列的右边缘，鼠标变为 ↔ 拖拽图标时，按住左键左右拖动即可调整列宽。调整后的宽度自动保存，下次打开页面时保持不变。
- **列顺序调整**：按住任意列的表头，左右拖拽即可调整列的显示顺序。调整后的顺序自动保存。
- **排序**：点击带排序箭头的列头可升序/降序排列。
- **筛选**：列表顶部提供状态下拉筛选和关键词搜索。
- **分页**：表格底部分页栏支持切换每页显示条数（10/20/50/100 条）。

## 1. 整体导航

打开首页（`/`）顶部菜单有以下页面（自左向右）：

- **首页**（`/`） — 作战态势大盘（攻关单总数 / 进行中 / 已闭环 / 贡献总数 / 待审批提议；状态分布；Top 贡献人）+ 模块入口卡片。
- **攻关作战台**（`/attack`） — 攻关单可编辑表格（列表+CRUD+导出 Excel）。
- **荣誉殿堂**（`/honor`） — 贡献加权排行榜与个人贡献档案。
- **贡献录入**（`/contributions`） — 贡献记录可编辑表格。
- **导入**（`/import`） — Excel 导入（按 nodeType + identityKey upsert 增量导入）。
- **关系审批**（`/proposals`） — Hermes/启发式提议的候选关系审批队列（通过/拒绝）。
- **信息检索**（`/search`） — 跨攻关/贡献/关联的只读检索（Hermes 只读契约）。
- **发布包**（`/releases`） — 版本发布包元数据 + 下载链接登记。
- **权重文件**（`/weights`） — 模型权重文件元数据 + 存储链接登记。
- **攻关日报**（`/daily-report`） — 当日进展按攻关单分组的日报预览 + 复制到剪贴板。
- **跟催提醒**（`/reminders`） — 跟催/提醒队列（问题单跟催 / FE Deadline / CCB 提醒；当前为 stub 渠道）。
- **问题反馈**（`/bug-report`） — 提交系统问题和改进建议，支持截图上传和 Console 日志自动捕获。

每个页面上方都有页面标题；详情链接（如「关联全景」「找帮手」）会跳转到对应子页。

## 2. 攻关作战台（`/attack`）

### 浏览与过滤
- 表格按 `状态` 过滤栏（顶栏 `按状态过滤`）。回车应用过滤；清空恢复全部。
- 「导出 Excel」按当前 nodeType 全量导出（不受过滤影响）。
- 点「标题」单元格进入该攻关单的详情页（`/attack/:id`）。
- 点 ref 字段（如「当前处理人」）单元格直接跳到被引用实体的关联页（增量3c 的 ref-cell direct-jump）。

### 新建一行
- 点「新增行」→ 顶部出现该 nodeType 全部字段输入框（含中文 placeholder）→ 填要紧字段（必填如 `标题`/`状态`）→「创建」。

### 编辑一行
- 点该行「编辑」→ 单元格变 Input → 改 → 「保存」。

### 字段管理（运行时改 schema，零迁移）
- 每列表头有按钮：「改名」（改 label）、「退休」（数据保留，列消失）、「别名」（导入列归一）、「概念」（语义角色，跨 view 异名归并；如「负责人」）、「锚点」（共享最细粒度键，跨颗粒度派生关联；如「问题单号」）。
- 「+字段」按钮添加新字段（指定 name/label/type）。

### 攻关单详情（`/attack/:id`）
- Descriptions 列出所有非退休字段当前值。
- 「关联全景」链接 → `/related/attackTicket/:id`（增量3a 关联全景页）。
- **找帮手** 区块（增量5）：基于 KG 证据自动推荐"能帮的人"（共享问题单履历 + 贡献等级胜任度），点击人名跳到其关联页。
- 进展序列：「追加进展」（输入 + 按钮 → 增量追加一条 ProgressLog，附当前状态快照）；下方 Timeline 倒序展示历史。

## 3. 关联全景（`/related/:nodeType/:id`）

按"语义分组"列出本节点的关联：
- **按 concept 分组**（增量3b）：相同 concept（如「负责人」）的关联归到同一组。无 concept 的项仍按 nodeType 分组。
- **跨颗粒度（共享锚点）**（增量3d）：通过共享问题单号等锚点关联到的其它 nodeType（如同一问题单号下的攻关单+贡献+发布包+权重文件）。
- **候选关系（待审批）**（增量3c）：Hermes/启发式提议但未审批的候选（独立分组，标注置信度+理由）。

## 4. 贡献录入（`/contributions`）

与攻关作战台同形（可编辑表格）；字段：`贡献人`（ref→person）、`贡献类型`（enum 必填：发现/设计/实施/协调/公关）、`贡献等级`（普通/关键/核心）、`贡献描述`、`关联攻关单`（按攻关单号关联）、`周期`、`记录人`、`记录时间`。

## 5. 荣誉殿堂（`/honor`）

- 加权排行榜：按 `贡献等级` 加权（核心3/关键2/普通1）排序贡献人。
- 周期过滤：填入 `周期` 字段值（如 `2026-Q1`）后回车，仅显示该周期贡献的人。
- 点击「贡献人」名 → 个人贡献档案（`/honor/:name`），列出其全部贡献 + `关联攻关单`链接（如有）。

## 6. 导入（`/import`）

1. 选「导入类型」（攻关单 / 贡献记录 / 发布包 / 权重文件 / 人员）。
2. 「选择 Excel 文件」上传 `.xlsx`。
3. 提示「导入新增 N · 已更新 M」——按 identityKey upsert：已存在的 identityKey 行**更新**（属性合并），其他**新建**（增量8）。再次导入相同表不会重复。

字段映射：Excel 列名匹配 schema 的 `name`/`label`/`aliases` 任一即可。`required` 校验不通过的行被跳过。

## 7. 关系审批（`/proposals`）

- 点「扫描候选」：后端启发式 proposer（增量3c）按规范化人名 Levenshtein≤1 提议 `SAME_AS` 候选（疑似同一人的不同写法）。
- 表格列：来源/目标实体 id、关系（`SAME_AS`）、置信度、理由（如「张伟≈张玮 dist=1」）。
- 「通过」→ 走结构化权威合并（§2.1 person 合并：边迁移+属性合并+审计+不可逆），候选→已通过。
- 「拒绝」→ 候选→已拒绝；后续 scan 同三元组不重复出现。

## 8. 信息检索（`/search`）

- 输入关键词→回车。后端跨全部 nodeType 做属性子串大小写不敏感检索（增量4）。
- 结果列「summary（nodeType）」，点击跳到对应详情页或关联页。
- 严格只读（Hermes 只读契约；任意 SDK/MCP 适配器可包装该 HTTP 契约接入 Hermes）。

## 9. 发布包 / 权重文件（`/releases`、`/weights`）

元数据 + 链接登记（增量7；§13#6 已决定不做文件托管）：
- 发布包字段：`版本号`（必填）/ `产品` / `发布日期` / `下载链接` / `责任人`（ref→person, concept=负责人）/ `关联问题单`（anchor=问题单号；与攻关单/贡献跨颗粒度联动）/ `描述` / `备注`。
- 权重文件字段：`名称`（必填）/ `模型` / `存储链接` / `责任人` / `训练日期` / `关联问题单` / `备注`。
- 同问题单号下的攻关单 + 发布包 + 权重在彼此关联全景页的「跨颗粒度（共享锚点）」组互见。

## 10. 攻关日报（`/daily-report`）

- 选日期（默认今日）→ 自动从 ProgressLog 派生当日"按 攻关单 分组"的进展+摘要（增量9）。
- 「复制到剪贴板」→ 中文纯文本日报，粘到任意 IM/邮件即可（外发渠道接入前的过渡用法）。

## 11. 跟催提醒（`/reminders`）

- 点「扫描提醒」→ 后端规则引擎生成（增量10/11）：
  - **问题单跟催**：状态 ∈ {待响应/处理中/进行中} 且 ≥3 天无进展 → 通知当前处理人。
  - **FE Deadline 提醒**：`客户要求解决时间` 在未来 ≤3 天 + 状态未闭环 → 通知当前处理人。
  - **CCB 提醒**：攻关单 `是否需CCB=是` + 状态未闭环 → 通知上 CCB 评审。
- 表格每行「发送(stub)」「忽略」按钮。**当前 stub 渠道：点击发送仅标记已发送+审计，不真发**；接入 SMTP/eSpace/welink 后真发。
- 7 天幂等：同（kind, ticketId, 收件人）7 天内不重复生成。

## 12. 进展更新 = 唯一权威写路径

任何字段修改、新建、删除、合并、审批，最终都经"结构化模型 + 审计"。所有变化都有 `audit_log` 记录（CREATE/UPDATE/DELETE/PROGRESS/MERGE/审批）。
