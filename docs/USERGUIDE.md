# 作战管理工具 使用手册

> 本手册面向运营团队的日常用户，无需了解技术细节，按照本手册即可完成所有操作。

---

## 快速开始

**访问地址**：`http://[服务器IP]:5173`（由管理员提供具体 IP 地址）

第一次打开后，会看到顶部导航菜单和首页大盘（Homepage）。大盘展示当前攻关单总数、进展条数、待审批提议数等汇总信息。

## 功能详解

### 1. 攻关作战台（`/attack`）

攻关作战台是本工具的核心模块，用于追踪所有线上攻关事件。

#### 1.1 查看攻关单

进入 `/attack` 后，默认显示全部攻关单列表。可以：

- 点击列标题排序
- 使用顶部过滤器按**状态**筛选（待响应 / 处理中 / 进行中 / 已解决 / 已关闭）
- 点击「标题」跳转到攻关单详情页

#### 1.2 创建攻关单

点击表格右上角「+ 新建」按钮，填写：

| 字段 | 说明 | 必填 |
|---|---|---|
| 标题 | 攻关单简要描述 | 是 |
| 状态 | 初始状态（通常选「待响应」） | 是 |
| 问题单号 | 如 PB-12345（用于跨单关联） | 否 |
| 事件级别 | P1 / P2 / P3 / P4A（影响 SLA 上升） | 否 |
| 当前处理人 | 填写人员姓名（自动关联 person 节点） | 否 |
| 客户名称 | 相关客户 | 否 |
| 客户要求解决时间 | 截止时间 | 否 |

点击「保存」后，攻关单自动生成唯一 ID，并进入列表。

#### 1.3 编辑攻关单

在列表页双击单元格可**内联编辑**，修改后点击空白处或回车确认。

或进入详情页（点击标题）后点击「编辑」按钮，修改全部字段后保存。

#### 1.4 状态流转

状态变更有两种方式：

**方式一（内联）**：在列表页直接点击「状态」列的枚举值下拉框选择新状态。

**方式二（详情页）**：进入详情页后，点击「流转状态」按钮，选择目标状态，可输入流转备注。每次流转会自动在进展时间线追加一条记录。

状态流转路径：`待响应 → 处理中 → 进行中 → 已解决 → 已关闭`

#### 1.5 追加进展

在详情页（`/attack/:id`）可以：

1. 在「进展记录」区域点击「+ 追加进展」
2. 填写进展内容和当前状态快照
3. 保存后该条进展永久保留（不可删除）

进展记录按时间顺序排列，完整呈现攻关历程。

#### 1.6 找帮手

在攻关单详情页点击「找帮手」，系统会根据以下依据推荐可协助的人员：
1. 曾处理过相同问题单号（PB-XXXXX）的人
2. 在相关问题单下有核心/关键贡献记录的人

#### 1.7 查看关联全景

在攻关单详情页点击「关联全景」，或直接访问 `/related/attackTicket/:id`，可以看到：
- 该攻关单通过 ref 字段关联的人员
- 同一问题单号下的其他攻关单（通过锚点共享）
- 与该单有冲突/重叠关系的其他攻关单

### 2. 问答助手（`/hermes`）

Hermes 是内置的自然语言问答助手，直接用中文提问，即可获取数据汇总和关联分析。

#### 2.1 支持的问题类型

| 问法示例 | 返回内容 | 数据展示形式 |
|---|---|---|
| `PB-12345 涉及哪些攻关单？` | 该问题单下所有攻关单列表 | 表格 |
| `断网故障 谁负责？` | 匹配标题的攻关单及负责人 | 表格 |
| `断网故障 现在状态？` | 匹配标题的攻关单状态+最新进展 | 卡片 |
| `谁最忙？` | 当前活跃攻关单最多的处理人排名 | 统计图 |
| `张三 贡献了什么？` | 张三的贡献记录列表 | 表格 |
| `今天谁动了什么？` | 今天有进展更新的攻关单 | 表格 |
| `本周有哪些变动？` | 本周有进展更新的攻关单 | 表格 |
| `断网故障 找谁帮忙？` | 推荐可协助处理该攻关单的人员 | 卡片列表 |
| 任意关键词 | 全文检索所有数据 | 表格 |

#### 2.2 如何提问

1. 在问答框输入问题（中文），按回车或点击「提问」
2. 右侧显示文字答案
3. 下方自动渲染对应的数据视图（表格/卡片/统计）

#### 2.3 固定常用问答

对于需要反复查看的数据（如"今天谁动了什么"），可点击答案旁的「📌 固定」按钮，将该问答结果固定到侧边栏。

- 固定后的 UI 在每次打开 Hermes 页面时自动刷新展示
- 可对固定的 UI 重命名（点击标题旁的编辑图标）
- 点击「取消固定」移除

### 3. 荣誉殿堂（`/honor`）

荣誉殿堂记录团队成员的贡献，并生成排行榜。

#### 3.1 贡献等级与积分

| 等级 | 分值 | 适用场景 |
|---|---|---|
| 普通 | 1 分 | 常规参与 |
| 关键 | 3 分 | 关键路径或较难问题 |
| 核心 | 8 分 | 核心突破或高影响贡献 |

**注意**：贡献等级「关键」和「核心」的标定仅 Leader 及以上权限可操作。

#### 3.2 查看排行榜

- 默认展示个人总积分排名
- 可按「周期」过滤（如填写 `2026Q2`）
- 点击「按团队汇总」切换为团队维度排名
- 点击人员名字进入个人档案页

#### 3.3 个人档案（`/honor/:name`）

展示该人员的所有贡献记录，以及每条贡献关联的攻关单。

#### 3.4 新建贡献记录

进入 `/contributions` 点击「+ 新建」，填写：

| 字段 | 说明 | 必填 |
|---|---|---|
| 贡献人 | 人员姓名 | 是 |
| 贡献等级 | 普通 / 关键 / 核心 | 是（Leader+ 才能标定关键/核心） |
| 贡献类型 | 如"问题解决"、"文档沉淀"等 | 否 |
| 贡献描述 | 详细说明 | 否 |
| 关联攻关单 | 填写攻关单号（如 AT-001）或标题，自动关联 | 否 |

---

### 4. 责任矩阵（`/responsibility`）

责任矩阵以 Mermaid 流程图形式展示：

1. **SLA 升级规则**：不同事件级别超时后升级给哪个角色（来自系统配置）
2. **人员负责关系**：谁负责哪些攻关单（来自 ASSIGNED_TO / ESCALATED_TO 边）
3. **冲突关系**：攻关单之间的冲突（虚线表示）

图可直接在页面缩放和截图。

---

### 5. 关联全景（`/related/:nodeType/:id`）

关联全景视图展示一条记录的所有关联：

- **出向关联**：本记录引用了哪些其他记录（如"当前处理人"字段引用的 person）
- **入向关联**：哪些其他记录引用了本记录
- **共锚关联**：通过共同的问题单号 / 事件单号关联的其他攻关单
- **冲突关联**：与本记录有冲突/重叠关系的记录
- **手工关联线**：人工标注的关联关系（带备注）
- **候选提议**：系统自动发现但待确认的潜在关联

可通过「调整深度」按钮展开多跳关联（最多 5 跳）。

### 6. 表结构管理（`/schema-wizard`）

不需要工程师介入，业务人员即可动态调整表的字段。

#### 6.1 给现有表新增字段

1. 进入 `/schema-wizard`
2. 在左侧选择目标表（如"攻关单"）
3. 点击「+ 新增字段」
4. 填写字段名、类型（文字/数字/日期/枚举/引用）、显示标题
5. 保存后**立即生效**（无需重启，无需数据库迁移）

#### 6.2 字段操作

| 操作 | 说明 |
|---|---|
| 重命名显示标题 | 改 UI 中的列标题，不影响已有数据 |
| 设置别名 | 导入 Excel 时多个列名映射到同一字段 |
| 停用字段（retire） | 字段隐藏但数据保留，可随时恢复 |
| 设置概念 | 标记语义（如"负责人"），用于跨表关联 |
| 设置锚点 | 字段值作为共享锚点，同值记录自动关联 |

#### 6.3 新建业务表

1. 点击「+ 新建表」
2. 填写表名（英文 camelCase）和中文显示名
3. 添加至少一个字段
4. 保存后系统自动生成 JSON 配置并注册
5. 前端需要刷新页面后在菜单中看到新表（需管理员添加导航入口）

### 7. 数据导入（`/import`）

支持从现有 Excel 表格批量导入数据。

#### 7.1 导入步骤

1. 进入 `/import`
2. 在「目标类型」下拉框选择要导入到哪个业务表（如"攻关单"）
3. 点击「选择文件」上传 Excel（.xlsx 格式）
4. 勾选「仅预览（不写入）」可先看导入计划（哪些行会新建、哪些更新、哪些跳过）
5. 取消预览勾选，点击「导入」执行

#### 7.2 Excel 格式要求

- 第一行为列标题
- 列名需与业务表的字段名、显示标题或别名**之一**完全匹配（支持中英文）
- 攻关单导入时通过「攻关单号」字段去重（已存在则更新，不存在则新建）

#### 7.3 导入结果说明

| 结果 | 含义 |
|---|---|
| 新建 (create) | 数据库中不存在，新增一条记录 |
| 更新 (update) | 通过唯一标识匹配到已有记录，合并更新字段 |
| 跳过 (skip) | 该行数据不满足必填字段或格式校验 |

### 8. 跟催提醒（`/reminders`）

系统自动扫描超期未响应的攻关单，生成跟催提醒。

#### 8.1 查看提醒

进入 `/reminders`，默认显示「待发送」状态的提醒。可按状态过滤：待发送 / 已发送 / 已忽略。

每条提醒包含：提醒类型、关联攻关单、收件人、提醒内容。

#### 8.2 处理提醒

- 点击「发送」：标记为已发送，并通过配置的渠道通知收件人
- 点击「忽略」：标记为已忽略，本轮不再发送

#### 8.3 手动触发扫描

系统每次执行「定时任务」（`POST /api/jobs/tick`）时自动扫描。也可在页面点击「扫描提醒」手动触发。

---

### 9. 自动日报（`/daily-report`）

汇总当天所有有进展记录的攻关单，生成标准格式日报。

- 默认显示当天日报；可通过日期选择器切换历史日期
- 每个攻关单显示：标题、当前状态、当天所有进展条目
- 汇总数据：当天涉及攻关单数、进展条目总数、各状态分布

**发布日报**：点击「发布日报」按钮，系统会对当天有进展的攻关单各自的「日报发布数量」字段 +1，并记录审计日志。

## 常见问题

**Q：数据导入后字段全是空的怎么办？**

A：Excel 的列标题必须与系统字段的「名称」、「显示标题」或「别名」完全匹配（包括大小写和空格）。可先在 `/schema-wizard` 查看字段别名，或在导入前给 Excel 列标题加别名映射。

---

**Q：找不到某个攻关单怎么办？**

A：使用全文检索（`/search`）输入攻关单标题片段、问题单号、处理人姓名等任意关键词搜索。也可以问 Hermes（`/hermes`）直接用自然语言描述。

---

**Q：如何关联两条不同类型的数据？**

A：有三种方式：
1. **自动关联**：如果两条记录的字段有 `anchor`（如相同的问题单号），系统会自动建立锚点关联，在关联全景中可见。
2. **手工关联**：在关联全景页面（`/related/...`）点击「+ 手工关联」，选择目标记录并填写关联备注。
3. **提议审批**：系统自动检测相似记录并生成提议，在 `/proposals` 页面审批通过后自动建立关联。

---

**Q：如何查看某人处理了哪些攻关单？**

A：方法一：问 Hermes「张三 负责哪些攻关单」；方法二：在攻关单列表过滤「当前处理人=张三」；方法三：查看荣誉殿堂的个人档案 `/honor/张三`。

---

**Q：状态流转出错怎么办？**

A：攻关单状态只能设置为系统配置的合法值（待响应 / 处理中 / 进行中 / 已解决 / 已关闭）。如果当前状态不在合法值内，先通过编辑直接修改「状态」字段，再使用状态流转功能。

---

**Q：不小心删除了数据怎么恢复？**

A：在 `/audit` 审计日志中找到删除记录（action=DELETE），查看 `changes` 字段中保存的数据快照，联系管理员手动恢复。当前版本不提供自动撤销功能。

---

**Q：邮件发不出去怎么办？**

A：进入 `/email`，先检查 SMTP 配置是否正确（服务器地址、端口、用户名、密码、发件人邮箱），然后用「发送测试邮件」按钮验证连接是否正常。

---

**Q：为什么某个字段无法修改贡献等级？**

A：贡献等级「关键」和「核心」仅 Leader 及 Admin 权限可设置。普通用户只能设置为「普通」。如需标定更高等级，请联系 Leader 操作，或让 Leader 以 Leader 身份操作。

---

**Q：如何批量导出数据？**

A：在任意业务表列表页（如攻关作战台），点击右上角「导出 Excel」按钮，即可下载当前表所有记录的 Excel 文件。

---

**Q：「合并人员」有什么风险？**

A：人员合并操作**不可逆**，执行前会展示预览（字段合并情况 + 受影响的边数量），请仔细确认后再执行。合并后 from 节点删除，所有关联迁移到 to 节点。如有疑虑请联系管理员。
