# v2.7.0 Release — Hermes 体验收尾 + Schema-as-UI 全栈化 + 多视图

- 日期: 2026-06-01
- 起点: master @ 22cdddf (v2.6.0)
- 三桶并行: r-hermes-polish / r-schema-all / r-views + integ-v2.7 集成

## 一句话总结

把 v2.6 几条主线**贯穿落地**：Schema-as-UI 从 1 个 nodeType 推到 7 个、视图从 1 种推到 4 种、LLM golden set 从 14/15 提升到 15/15、部署 systemd drop-in 从手工到自愈。

## 三桶交付

### ① Hermes 体验收尾 (5 commits)

| 项 | 落地 |
|---|---|
| GET /api/llm-settings/models | 动态拉取 provider 实际可用 model 列表(`/v1/models`) |
| UI defaultModel/smallModel | AutoComplete → 动态 Select + 「刷新模型列表」按钮 |
| /api/llm-settings/test env-fallback | admin 未存 DB 也能用 env 实测连接 |
| Hermes prompt 优化 | 审计追溯类问题优先调 `get_audit(actor='X')` |
| Golden set | 15/15 通过(v2.6 是 14/15,Q7 修复) |
| deploy-direct.mjs 自愈 | 自动检测/备份冲突 systemd drop-in;`--keep-old-drop-ins` 关闭 |
| 默认 model | `glm-4-flash` 智谱免费层,零成本 |

### ② Schema-as-UI 全栈化 (13 commits)

| 项 | 落地 |
|---|---|
| 7 个 schemas/*.json | contribution / teamContribution / person / helpRequest / bugReport / proposal / reminder 全部补 group/order/specialControl |
| virtual schema 概念 | 4 个虚拟 nodeType(helpRequest/bugReport/proposal/reminder)用 `virtual:true` 标记,后端 `/api/nodes/<虚拟类型>` gate 400 防双写 |
| useSchema hook | 通用 schema 加载 |
| SchemaFormBody / SchemaViewBody | 按 group 自动 Divider/Card 渲染 |
| 7 个抽屉 schema 驱动 | PeopleList / Contributions(4 抽屉) / HelpCenter / BugReport / Proposals / Reminders 全部移除硬编码 Form.Item |
| e2e | schema-driven-all.spec.ts 覆盖 5 nodeType + virtual gate |

### ③ 多视图 (4 commits)

| 视图 | URL | 实现要点 |
|---|---|---|
| 表格 | `?view=table`(默认) | v2.6 已有 |
| 看板 | `/attack?view=kanban` | HTML5 native DnD + Select 降级 + 乐观更新 + 失败回滚 |
| 日历 | `/attack?view=calendar` | antd `<Calendar fullscreen>` + 创建/解决时间 Switch + 色块 P0-P3 + Popover |
| 透视 | `/contributions?view=pivot` | 贡献人 × 类型 + 加权积分(3/2/1) + 颜色梯度 + 小计/总计 |

`AttackList` / `Contributions` 顶部加 `<Segmented>` 切换器,URL `?view=` 同步,切换不重置 filter / 搜索 / 分页。

## 经验沉淀

> v2.6 完成的是"用户可见的中期 UX 槽位填上"。v2.7 完成的是把这些主线**贯穿落地** — Schema-as-UI 从单点示范变成全栈通用、视图从只有 Table 到 4 种、LLM 从 14/15 到 15/15、部署从手工到自愈。这是把"组件化能力"转向"产品成熟度"的关键一版。配合 v2.6 的 LLM 直连 + UI 配置,**v2.7 之后管理员的全部日常运维都在浏览器里完成**:加字段、改 LLM、看通知、切视图,无需 SSH、无需重启。

下一步(v2.8 候选):Hermes 高级能力 — Memory(session 持续上下文)、Planner(多步骤规划)、Reflection(自检)、写工具开放(create/update 含审批流)、Hermes 公开 API。
