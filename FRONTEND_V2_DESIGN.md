# 作战平台 v2 — 第一版设计规格

---

## 一、需求摘要

### 1.1 优先交付范围

| 模块 | 核心价值 |
|------|----------|
| **攻关作战平台** | 发布、查看、搜索、管理攻关任务；条目化记录进展/人员/资源变化；求助网络 |
| **荣誉殿堂** | 团队/个人荣誉查询、排行榜、贡献档案 |
| **全员名单** | 人员信息管理（含邮件地址），管理员可导入 Excel |
| **求助系统** | 发送求助邮件含反馈链接，对方填写后自动录入系统 |
| **审计日志** | 全部操作留痕，可视化查看 |

### 1.2 渐进交付（后续）
信息检索、Hermes 问答、关系审批、冲突检测、KG 图谱、日报、邮件群组、
SLA 上升、自定义命令、Schema 管理等。

## 二、全局设计规范

### 2.1 技术栈
```
React 18 + Vite 6 + Ant Design 5 + react-router-dom 6 + TypeScript 5 (strict)
图标: @ant-design/icons
日期: dayjs
API: 原生 fetch 封装 → 124.156.193.122:3001
列拖拽: react-resizable + @dnd-kit/sortable
认证: JWT (7天过期) + bcrypt + RBAC角色
```

### 2.2 布局
```
┌──────────────────────────────────────────────────────┐
│ 顶部栏 48px: Logo + 用户 Dropdown (displayName + role + 登出) │
├────────┬─────────────────────────────────────────────┤
│ 侧边栏  │ 内容区 (max-width 1400px, 居中)              │
│ 200px  │                                             │
│ 可折叠  │                                             │
│ →64px  │                                             │
│        │                                             │
├────────┴─────────────────────────────────────────────┤
│ 公共反馈页（求助反馈等）使用独立空白布局，无侧边栏          │
└──────────────────────────────────────────────────────┘
```

### 2.3 色彩体系（Ant Design Token）
```
colorPrimary: #0050b3
colorSuccess: #52c41a
colorWarning: #faad14
colorError:   #ff4d4f
borderRadius: 6px
colorBgLayout: #f5f5f5
```

### 2.4 状态色（全局统一）
```ts
const STATUS_COLOR: Record<string, string> = {
  '待响应': 'gold',    '处理中': 'blue',
  '进行中': 'cyan',    '已解决': 'green',
  '已关闭': 'default', '高': 'red',
  '中': 'orange',      '低': 'blue',
};
const CONTRIBUTION_COLOR: Record<string, string> = {
  '核心': 'red',  '关键': 'orange', '普通': 'blue',
};
```

### 2.5 交互规范
- **新建/编辑**: 右侧 Drawer（width 520px），不用弹窗
- **删除**: Popconfirm，显示被删对象名称
- **列表**: 分页 20 条/页，支持排序
- **反馈**: message.success/error 顶部居中 2s
- **加载**: 列表页 Skeleton，详情页 Spin
- **空状态**: Ant Empty + 引导文案
- **破坏性操作**: 二次确认
- **语言**: 纯中文

## 五、求助系统 — 后端新增能力

### 5.1 数据模型

新增 `help_requests` 表（或用 nodeType=helpRequest + 专用表）：

```sql
CREATE TABLE IF NOT EXISTS help_requests (
  id TEXT PRIMARY KEY,
  ticket_id TEXT NOT NULL,          -- 关联攻关单
  requester_name TEXT NOT NULL,     -- 求助人
  target_name TEXT,                 -- 被求助人姓名
  target_email TEXT NOT NULL,       -- 被求助人邮箱
  category TEXT NOT NULL,           -- 环境/领域专家/团队协作/资源
  question TEXT NOT NULL,           -- 求助内容
  extra_note TEXT,                  -- 附加说明
  feedback_token TEXT UNIQUE NOT NULL,  -- 反馈链接 token
  status TEXT NOT NULL DEFAULT '待回复',  -- 待回复/已回复
  feedback TEXT,                    -- 反馈内容
  feedback_by TEXT,                 -- 反馈人
  feedback_at TEXT,                 -- 反馈时间
  created_at TEXT NOT NULL,
  updated_at TEXT NOT NULL
);
```

### 5.2 新增 API

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/help-requests` | 创建求助 + 发送邮件 |
| GET | `/api/help-requests?ticketId=&status=` | 列表（可按攻关单/状态过滤） |
| GET | `/api/help/feedback/:token` | 公开页：获取求助信息（无需鉴权） |
| POST | `/api/help/feedback/:token` | 公开页：提交反馈（无需鉴权） |

### 5.3 邮件模板

```
主题: 【作战平台求助】{requester} 就「{ticket标题}」向您求助

{requester} 在攻关单「{ticket标题}」中需要您的帮助：

{question}

请点击以下链接回复：
http://124.156.193.122:3001/api/help/feedback/{token}

— 作战平台
```
